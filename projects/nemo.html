GitOps CI/CD & Infrastructure as Code

GitHub Actions → ArgoCD

Push to main triggers GitHub Actions build → Docker image pushed to ECR → ArgoCD detects Helm chart changes → auto-deploys to EKS cluster.

Helm + Kubernetes

All microservices (backend, frontend, AI, Redis, Kafka) managed via Helm charts with configurable values per environment.

Terraform IaC

AWS EKS cluster, VPC, node groups, and IAM roles provisioned via Terraform for reproducible infrastructure.

AI & Messaging Infrastructure

vLLM Model Serving

Deployed Gemma 3B/4B models via vLLM on GPU nodes, enabling high-throughput containerized inference for AI group creation and chatbot recommendation features.

Redis + Kafka

Redis for caching and session management. Kafka for async event streaming between microservices (AI processing queue, notification pipeline).

Monitoring & Load Testing

K6 Load Testing

Ran load tests with K6 to validate performance under concurrent users. Identified bottlenecks in AI inference pipeline.

SigNoz + Netdata

SigNoz for distributed tracing across microservices. Netdata for real-time server metrics and Discord alerting (which caught the scanning attack).

Real Production Incident
Scanning Attack on Production Server
2025.05.15 19:40 KST — V1 Phase (Public Subnet)
Detection

Netdata alert via Discord — 100% bad requests, attacker probing for .env files.

Discord alert from Netdata

Nginx logs — .env probe attempts

Resolution

Migrated all services to Private Subnet + WireGuard VPN. Only Nginx remained public-facing.

Result

Attack surface eliminated — zero scanning incidents after migration.

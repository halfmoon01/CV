Production Issue
File upload returns 403 — request never reaches backend
Users reported that uploading syllabus files (PDF) resulted in a 403 Forbidden error. The request was being blocked before hitting the origin server.
Detection

File upload triggered 403 Forbidden. Console showed repeated Failed to load resource: status 403 errors. Network tab confirmed CloudFront returning "Request blocked."

Console: 403 errors on PDF upload

Network: CloudFront "Request blocked"

Root Cause Analysis

1. CloudFront — HTTP Method Restriction
Default behavior only allowed GET / HEAD. File upload POST was rejected at the CDN layer.

2. WAF — SizeRestrictions_BODY Rule
AWSManagedRulesCommonRuleSet blocks request bodies exceeding 8KB. PDF uploads exceed this threshold.

Resolution

Fix 1: Updated CloudFront behavior for /api/* to allow GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE.

Fix 2: Changed SizeRestrictions_BODY from Block to Count.

CloudFront: Allowed HTTP Methods

WAF: SizeRestrictions_BODY → Count

Result

File uploads working correctly. PDF syllabus files now pass through CloudFront and WAF to reach the backend successfully.

V2 — Architecture Decision
Plan A: Private Subnet + Bastion Host

Considered isolating into a Private Subnet — requiring 3 EC2 instances + NAT Gateway.

Problem: Cost Explosion

Plan A: Private Subnet (3 EC2 + NAT GW)
  EC2 t2.small × 3               ~$51.00
  NAT Gateway ($0.045/hr × 730h) ~$32.85
  NAT Data Processing             ~$2-5
  CloudFront                      ~$1
  Route 53                        ~$0.50
  Total                        ~$88-91/month

NAT Gateway alone (~$32.85/month) exceeded the entire v1 budget. Not viable for a student project.

Plan B (Adopted): Single EC2 + CloudFront + WAF

Kept all containers on 1 EC2 and added CloudFront + WAF as the security layer:

  • CloudFront — hides origin EC2 IP
  • AWS WAF — blocks scanning patterns
  • Security Group — only accepts CloudFront IPs
Plan B: Single EC2 + WAF (Adopted)
  EC2 t2.small × 1               ~$17.00
  CloudFront                      ~$1
  Route 53                        ~$0.50
  WAF (Web ACL + rules)           ~$8
  Total                         ~$27/month

Plan A: 3 EC2 + NAT GW (~$88/mo)

Plan B: 1 EC2 + WAF (~$27/mo)

Result

Scanning bots blocked at WAF layer. EC2 no longer directly accessible. Cost stayed at ~$27/month — 70% savings vs Plan A.
